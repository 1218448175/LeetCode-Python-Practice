<h2><a href="https://leetcode.cn/problems/gas-station">134. 加油站</a></h2>
<h3>中等</h3>
<hr>
<p>在一条环路上有 <code>n</code> 个加油站，其中第 <code>i</code> 个加油站有汽油 <code>gas[i]</code><em> </em>升。</p>
<p>你有一辆油箱容量无限的的汽车，从第<em> </em><code>i</code><em> </em>个加油站开往第<em> </em><code>i+1</code><em> </em>个加油站需要消耗汽油 <code>cost[i]</code><em> </em>升。你从其中的一个加油站出发，开始时油箱为空。</p>
<p>给定两个整数数组 <code>gas</code> 和 <code>cost</code> ，如果你可以按顺序绕环路行驶一周，则返回出发时加油站的编号，否则返回 <code>-1</code> 。如果存在解，则 <strong>保证</strong> 它是 <strong>唯一</strong> 的。</p>
<p> </p>
<p><strong>示例 1:</strong></p>
<pre><strong>输入:</strong> gas = [1,2,3,4,5], cost = [3,4,5,1,2]
<strong>输出:</strong> 3
<strong>解释:
</strong>从 3 号加油站(索引为 3 处)出发，可获得 4 升汽油。此时油箱有 = 0 + 4 = 4 升汽油
开往 4 号加油站，此时油箱有 4 - 1 + 5 = 8 升汽油
开往 0 号加油站，此时油箱有 8 - 2 + 1 = 7 升汽油
开往 1 号加油站，此时油箱有 7 - 3 + 2 = 6 升汽油
开往 2 号加油站，此时油箱有 6 - 4 + 3 = 5 升汽油
开往 3 号加油站，你需要消耗 5 升汽油，正好足够你返回到 3 号加油站。
因此，3 可为起始索引。</pre>

## 题解：贪心算法实现 $O(n)$ 线性扫描

### 核心设计思想

这道题的核心矛盾在于：如何在不逐个尝试每个起点（避免 $O(n^2)$ 复杂度）的情况下，快速找到那个唯一的合法起点。

核心思路基于以下两个数学推论：

1. **总和判定**：如果 $\sum \text{gas} < \sum \text{cost}$，说明总油量小于总消耗，无论从哪出发都无法环绕一周。

2. **局部失败推论**：如果在 $i$ 站出发，最远只能到达 $j$ 站（在前往 $j+1$ 的途中油量耗尽），那么 **$i$ 到 $j$ 之间的任何一个车站**都不可能作为起点。

本题代码采用了一种**动态位移**的策略：利用一个变量维护从当前候选起点出发后的剩余油量，一旦“负债”，则立即跳过整个失败区间寻找新起点。

---

### 算法流程拆解

#### 1. 第一步：全局合法性检查

- 直接对比 `sum(gas)` 和 `sum(cost)`。

- 若总油量不足，直接返回 **-1**。这一步保证了只要代码继续往下走，就一定存在解。

#### 2. 第二步：贪心寻找断点与起点更新

- 设置变量 `total_spare` 记录从当前候选起点出发后的**累计剩余油量**。

- 设置 `min_index` 记录最后一个导致“油量破产”的站点索引。

- **正向遍历**数组：
  
  - 计算当前站点的净增益 `spare = gas[i] - cost[i]` 并累加到 `total_spare`。
  
  - **判断逻辑**：如果 `total_spare < 0`，说明从当前候选起点到站点 $i$ 这段路程是“不可逾越”的。
  
  - **重置操作**：将 `total_spare` 清零（表示从下一站重新开始计算），并将 `min_index` 更新为当前索引 $i$。

#### 3. 第三步：结果映射

- 由于最后一次让 `total_spare` 跌破 0 的位置是 `min_index`，那么唯一的合法起点必然是 `min_index + 1`。

- 通过 `(min_index + 1) % n` 处理边界情况（如果所有站点都让油量增加，起点即为 0）。

---

### 复杂度分析

- **时间复杂度**：$O(n)$。代码进行了两次求和运算（通常 `sum()` 是 $O(n)$）以及一次 $O(n)$ 的线性遍历，整体依然是线性时间。

- **空间复杂度**：$O(1)$。只使用了 `min_index` 和 `total_spare` 等常数级辅助变量，没有使用额外的数组或哈希表。
