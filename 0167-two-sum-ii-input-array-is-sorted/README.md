## [167. 两数之和 II - 输入有序数组](https://leetcode.cn/problems/two-sum-ii-input-array-is-sorted/)

### 中等

给你一个下标从 **1** 开始的整数数组 `numbers` ，该数组已按 **非递减顺序排列**  ，请你从数组中找出满足相加之和等于目标数 `target` 的两个数。如果设这两个数分别是 `numbers[index1]` 和 `numbers[index2]` ，则 `1 <= index1 < index2 <= numbers.length` 。

以长度为 2 的整数数组 `[index1, index2]` 的形式返回这两个整数的下标 `index1` 和 `index2`。

你可以假设每个输入 **只对应唯一的答案** ，而且你 **不可以** 重复使用相同的元素。

你所设计的解决方案必须只使用常量级的额外空间。

<p><strong class="example">示例 1：</strong></p>
<pre><strong>输入：</strong>numbers = [<strong><em>2</em></strong>,<strong><em>7</em></strong>,11,15], target = 9
<strong>输出：</strong>[1,2]
<strong>解释：</strong>2 与 7 之和等于目标数 9 。因此 index<sub>1</sub> = 1, index<sub>2</sub> = 2 。返回 [1, 2] 。</pre>

---

## 题解：双指针缩减搜索空间

### 1. 核心思路：利用有序性

如果是一个无序数组，我们通常需要哈希表来记录数值。但题目给出的数组是**非递减排列**的，这为我们提供了方向感：

- 如果我们让左端的较小数和右端的较大数相加，结果太大，就必须把右端的指针向左移（寻找更小的数）。

- 如果结果太小，就必须把左端的指针向右移（寻找更大的数）。

这种**“夹逼”**的思想，能够确保我们在 $O(n)$ 的时间内找到目标，且不需要额外的存储空间。

---

### 2. 逻辑步骤拆解

- **初始化指针**：设置左指针 `l` 指向数组起始位置，右指针 `r` 指向数组末尾。

- **循环判断**：
  
  - 计算当前两指针指向数值之和 `total`。
  
  - **恰好命中**：如果 `total == target`，说明找到了目标。由于题目要求返回下标从 **1** 开始，所以返回 `[l + 1, r + 1]`。
  
  - **和偏小**：如果 `total < target`，说明需要更大的数值，因此将左指针 `l` 右移一位。
  
  - **和偏大**：如果 `total > target`，说明需要更小的数值，因此将右指针 `r` 左移一位。

- **边界处理**：虽然题目保证了有唯一解，但末尾返回 `[-1, -1]` 增强了代码的健壮性。

---

### 3. 为什么这个算法是正确的？（缩减空间证明）

很多初学者会担心：*“当我移动左指针时，会不会错过右指针之前对应的某个解？”*

- 实际上，由于数组是有序的，如果你移动了左指针（说明当前和太小），那么当前左指针指向的数与右指针右侧任何更大的数相加，显然都会超过 `target`。

- 换句话说，每一次指针的移动，都等同于排除了一整行或一整列不可能的组合，直到锁定唯一答案。

---

### 4. 复杂度分析

- **时间复杂度**：$O(n)$
  
  其中 $n$ 是数组长度。两个指针总共移动次数不会超过 $n$ 次。

- **空间复杂度**：$O(1)$
  
  只使用了 `l`、`r` 和 `total` 几个常数级别的变量，完美符合题目要求的“常量级额外空间”。
