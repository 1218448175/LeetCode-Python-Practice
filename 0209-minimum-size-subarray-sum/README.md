## [209. 长度最小的子数组](https://leetcode.cn/problems/minimum-size-subarray-sum/)

### 中等

给定一个含有 `n` 个正整数的数组和一个正整数 `target` **。**

找出该数组中满足其总和大于等于 `target` 的长度最小的 **子数组** `[numsl, numsl+1, ..., numsr-1, numsr]` ，并返回其长度**。**如果不存在符合条件的子数组，返回 `0` 。

<p><strong>示例 1：</strong></p>
<pre><strong>输入：</strong>target = 7, nums = [2,3,1,2,4,3]
<strong>输出：</strong>2
<strong>解释：</strong>子数组 <code>[4,3]</code> 是该条件下的长度最小的子数组。
</pre>

---

## 题解：滑动窗口与动态区间压缩策略

### 1. 核心思路：滑动窗口

问题的本质是在连续序列中寻找满足特定条件的“最短区间”。相比于暴力枚举所有子数组（$O(N^2)$），滑动窗口通过动态调整左右边界，将复杂度优化至线性：

- **窗口扩张（右边界 `right`）**：我们不断向右移动右指针，将元素加入当前 `total` 中，直到窗口内的总和满足 `>= target`。

- **窗口收缩（左边界 `left`）**：一旦条件满足，我们不再盲目向右，而是尝试通过移动左指针来“压缩”窗口，在保持条件成立的前提下寻找更短的长度。

- **实时更新**：在每次窗口收缩的过程中，实时更新最小长度 `ans`。

---

### 2. 执行逻辑的详细拆解

代码通过一个 `for` 循环和嵌套的 `while` 循环实现了窗口的平滑移动：

#### A. 寻找可行解（扩张）

遍历数组，将 `nums[right]` 累加到 `total`。这相当于在探索以 `right` 结尾的可能满足条件的子数组。

#### B. 优化最优解（收缩）

当 `total >= target` 时，说明当前窗口是一个“可行解”。但由于我们要找的是“最小长度”，因此开启收缩逻辑：

1. **记录当前长度**：计算 `right - left + 1` 并与全局最小值 `ans` 比较。

2. **移除左侧元素**：从 `total` 中减去 `nums[left]`。

3. **指针右移**：`left += 1`，尝试缩减规模。
   
   只要 `total` 依然满足 `>= target`，这个过程就会持续，确保捕捉到每一组可能的最小区间。

#### C. 无解处理

初始化 `ans` 为 `n + 1`（一个不可能达到的最大值）。如果遍历结束后 `ans` 依然没变，说明整个数组之和都小于 `target`，按照题目要求返回 `0`。

---

### 3. 逻辑亮点：双指针的非回溯性

滑动窗口之所以高效，是因为左右指针都**只向右移动，从不回头**：

- **避免重复计算**：我们利用了“正整数数组”的特性，即窗口右移和必增，左移和必减。

- **单向演进**：每个元素最多被 `right` 访问一次，被 `left` 移除一次。这种“毛毛虫式”的移动方式保证了逻辑的严丝合缝。

---

### 4. 复杂度分析

- **时间复杂度**：$O(N)$。虽然代码中有嵌套循环，但每个元素在整个过程中最多进出窗口各一次，因此总操作次数与数组长度 $N$ 成线性关系。

- **空间复杂度**：$O(1)$。我们只使用了常数个额外变量（`left`, `total`, `ans`）来存储状态，与输入规模无关。
