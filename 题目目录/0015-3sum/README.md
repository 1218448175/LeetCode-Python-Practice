## [15. 三数之和](https://leetcode.cn/problems/3sum/)

### 中等

给你一个整数数组 `nums` ，判断是否存在三元组 `[nums[i], nums[j], nums[k]]` 满足 `i != j`、`i != k` 且 `j != k` ，同时还满足 `nums[i] + nums[j] + nums[k] == 0` 。请你返回所有和为 `0` 且不重复的三元组。

**注意**：答案中不可以包含重复的三元组。

<p><strong>示例 1：</strong></p>
<pre><strong>输入：</strong>nums = [-1,0,1,2,-1,-4]
<strong>输出：</strong>[[-1,-1,2],[-1,0,1]]
<strong>解释：</strong>
nums[0] + nums[1] + nums[2] = (-1) + 0 + 1 = 0 。
nums[1] + nums[2] + nums[4] = 0 + 1 + (-1) = 0 。
nums[0] + nums[3] + nums[4] = (-1) + 2 + (-1) = 0 。
不同的三元组是 [-1,0,1] 和 [-1,-1,2] 。
注意，输出的顺序和三元组的顺序并不重要。
</pre>

---

## 题解：排序预处理与双指针

### 1. 核心思路：排序 + 双指针

“三数之和”的本质是在 $O(N^3)$ 的暴力枚举基础上进行降维打击。为了高效地找到不重复的三元组，我们采取了以下核心策略：

- **排序为基石**：先对数组进行升序排列。排序的作用有二：一是方便通过双指针根据和的大小移动方向；二是让重复的数字相邻，便于去重。

- **固定一数，搜索两数**：遍历数组，固定当前的 `nums[i]` 作为第一个数。随后在 `i` 之后的区间 `[i+1, n-1]` 内寻找另外两个数 `nums[j]` 和 `nums[k]`。

- **双指针移动（夹逼法）**：由于数组已排序，如果 `nums[i] + nums[j] + nums[k] > 0`，说明和太大，需将右指针 `k` 左移；反之则将左指针 `j` 右移。

---

### 2. 三个关键节点的去重逻辑

为了满足题目“不包含重复三元组”的要求，代码在不同阶段实施了严格的去重：

#### A. 第一层循环去重（固定位 `i`）

在遍历 `i` 时，如果当前数字与前一个数字相同（`nums[i] == nums[i-1]`），直接跳过。这保证了我们不会以同一个数字作为三元组的开头进行重复搜索。

#### B. 第二层循环去重（左指针 `j`）

同理，在固定了 `i` 后，如果 `j` 的值与前一个 `j` 相同，也会跳过。这避免了在同一个起始位置下，选中相同的第二个数。

#### C. 右指针的动态收缩（右指针 `k`）

代码中使用 `while` 循环在 `nums[j] + nums[k]` 过大时不断减小 `k`。这种方式比简单的三重循环更高效，因为它利用了单调性，将 $O(N^2)$ 的后两层枚举压缩到了 $O(N)$。

---

### 3. 逻辑亮点：双指针的同步演进

你的代码逻辑中，`j` 是从左向右移动的，而 `k` 在整个 `j` 的循环过程中是**从右向左单调回退**的：

- **无需重置 `k`**：对于同一个 `i`，随着 `j` 的增加（数值变大），满足条件的 `k` 必然只能保持不变或减小。因此，`k` 不需要每次都重置到 `n-1`，这一步将整体复杂度锁定在了 $O(N^2)$。

- **相遇即止**：当 `j == k` 时，说明当前 `i` 的所有可能性已穷尽，直接 `break` 进入下一个 `i` 的循环。

---

### 4. 复杂度分析

- **时间复杂度**：$O(N^2)$。
  
  - 排序耗时 $O(N \log N)$。
  
  - 外层循环遍历 $N$ 次，内层双指针 `j` 和 `k` 累计移动次数也是 $O(N)$ 级别。
  
  - 综合得 $O(N^2)$，远优于暴力解法的 $O(N^3)$。

- **空间复杂度**：$O(\log N)$。
  
  - 这主要取决于排序算法所消耗的栈空间。如果不计算返回结果所需的存储空间，额外辅助空间极低。
