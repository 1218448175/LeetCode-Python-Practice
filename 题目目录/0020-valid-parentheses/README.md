## [20. 有效的括号](https://leetcode.cn/problems/valid-parentheses/)

### 简单

给定一个只包括 `'('`，`')'`，`'{'`，`'}'`，`'['`，`']'` 的字符串 `s` ，判断字符串是否有效。

有效字符串需满足：

1. 左括号必须用相同类型的右括号闭合。
2. 左括号必须以正确的顺序闭合。
3. 每个右括号都有一个对应的相同类型的左括号。

**示例 1：**

<p><span class="example-io"><b>输入：</b>s = "()"</span></p>

## 题解：栈（Stack）的“后进先出”匹配策略

### 1. 核心思路：利用栈的对称性

括号匹配是一个典型的具有**对称性**和**嵌套性**的问题。最后进入的左括号，必须最先被匹配掉（即“后进先出”）。

你的代码采用了 **栈（Stack）** 这一数据结构来模拟这个过程：

- **遇到左括号**：说明遇到了一个等待闭合的“债”，将其入栈。

- **遇到右括号**：说明此时应该偿还最近的一个“债”。检查栈顶元素是否为对应的左括号。

- **最终状态**：所有字符处理完后，栈必须为空（债全部还清），且处理过程中不能出现匹配错误或无债可还的情况。

---

### 2. 执行逻辑的详细拆解

#### A. 映射关系维护

`patch_dict = {'}': '{', ')': '(', ']': '['}`

代码巧妙地使用右括号作为键（Key），对应的左括号作为值（Value）。这样在遇到右括号时，可以 $O(1)$ 时间获取它期望的“另一半”。

#### B. 匹配校验流程

遍历字符串 `s` 中的每个字符 `char`：

1. **左括号入栈**：如果是 `(`, `{`, `[` 其中之一，执行 `stack.append(char)`。

2. **右括号出栈校验**：
   
   - **栈不为空**：弹出栈顶元素 `left`。如果 `left` 与当前右括号不匹配（`patch_dict[char] != left`），则判定为无效，立即 `break`。
   
   - **栈为空**：说明当前是一个孤立的右括号，没有对应的左括号在等待，直接判定为无效。

#### C. 完备性检查

`if stack: ans = False`

即使遍历过程中没有报错，最后也需要检查栈内是否残留了没有匹配上的左括号（例如字符串为 `"([]"` 的情况）。只有栈彻底清空，才是完美的匹配。

### 4. 复杂度分析

- **时间复杂度**：$O(N)$。其中 $N$ 是字符串的长度。我们只需对字符串进行一次线性扫描，栈的入栈和出栈操作均为 $O(1)$。

- **空间复杂度**：$O(N + K)$。其中 $N$ 为栈在最坏情况下（全是左括号）存储的元素数，$K$ 为字典存储的字符集大小（常数级）。

---

### 5. 代码回顾

```python
class Solution:
    def isValid(self, s: str) -> bool:
        ans = True
        patch_dict = {'}': '{', ')': '(', ']': '['}
        stack = []

        for char in s:
            # 1. 遇到左括号，入栈等待
            if char in ('(', '{', '['):
                stack.append(char)
            # 2. 遇到右括号，尝试匹配
            elif stack:
                left = stack.pop()
                if patch_dict[char] != left: # 匹配失败
                    ans = False
                    break
            # 3. 栈为空却遇到右括号，非法
            else:
                ans = False
                break

        # 4. 检查是否有没有消掉的左括号
        if stack:
            ans = False

        return ans
```
