## [28. 找出字符串中第一个匹配项的下标](https://leetcode.cn/problems/find-the-index-of-the-first-occurrence-in-a-string/)

### 简单

<p>给你两个字符串 <code>haystack</code> 和 <code>needle</code> ，请你在 <code>haystack</code> 字符串中找出 <code>needle</code> 字符串的第一个匹配项的下标（下标从 0 开始）。如果 <code>needle</code> 不是 <code>haystack</code> 的一部分，则返回  <code>-1</code><strong> </strong>。</p>
<p> </p>
<p><strong class="example">示例 1：</strong></p>
<pre><strong>输入：</strong>haystack = "sadbutsad", needle = "sad"
<strong>输出：</strong>0
<strong>解释：</strong>"sad" 在下标 0 和 6 处匹配。
第一个匹配项的下标是 0 ，所以返回 0 。
</pre>

---

## 题解：KMP 算法（前缀函数法）

### 1. 核心思路：利用“对称性”跳过重复比较

KMP 算法的核心在于：当 `needle` 与 `haystack` 匹配过程中出现字符不统一时，主串的指针**不回退**，而是利用模式串 `needle` 自身的规律，将模式串向后“滑动”到最合适的位置。

这个规律被记录在 **Next 数组**（也称前缀函数）中。它记录了模式串中每一段子串的“最长相等前后缀长度”。这意味着，如果我们已经匹配了一段很长的部分，即便下一位错了，我们也知道前面有一部分前缀是可以直接复用的，不需要从头再来。

### 2. 逻辑拆解

#### 第一阶段：预处理（构建 Next 数组）

这一步的目的是研究 `needle` 字符串自己和自己的关系。

- **双指针协同**：使用一个快指针 `i`（后缀末尾）和一个慢指针 `j`（前缀末尾）。

- **匹配与回退**：如果 `i` 和 `j` 指向的字符相同，说明找到了更长的一段公共前后缀，`j` 向后移。如果不匹配，`j` 就根据之前已经计算好的 `next_arr` 结果进行回退，直到找到匹配的位置或退回起点。

- **记录结果**：将每个位置对应的 `j` 值存入数组，这个值代表了当前位置之前的字符串里，有多少位是对称的。

#### 第二阶段：搜索匹配（正式查找）

这一步是将 `needle` 放置在 `haystack` 上进行滑动。

- **指针移动**：主串指针 `i` 从头到尾只走一遍。模式串指针 `j` 则负责在 `needle` 上定位。

- **冲突处理**：当 `haystack[i]` 和 `needle[j]` 无法对上号时，我们不需要把 `i` 挪回起始点的下一位。相反，我们查看 `next_arr`，让 `j` 跳回到上一个可能匹配的对称点，继续比对。

- **成功判定**：当 `j` 的数值增加到与 `needle` 的总长度相等时，说明整个模式串都匹配上了。此时通过当前 `i` 的坐标减去模式串长度再加 1，就能反推出它在主串中出现的起始下标。

### 3. 复杂度分析

- **时间复杂度：$O(n + m)$**
  
  虽然代码里有嵌套的 `while` 循环，但主串指针 `i` 始终保持单向移动，不回溯。匹配和跳过的总操作次数与字符串长度呈线性关系。

- **空间复杂度：$O(m)$**
  
  我们需要额外创建一个长度为 $m$（模式串长度）的数组来存放跳跃查找的索引规律。
