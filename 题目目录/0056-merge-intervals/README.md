## [56. 合并区间](https://leetcode.cn/problems/merge-intervals/)

### 中等

以数组 `intervals` 表示若干个区间的集合，其中单个区间为 `intervals[i] = [starti, endi]` 。请你合并所有重叠的区间，并返回 *一个不重叠的区间数组，该数组需恰好覆盖输入中的所有区间* 。

**示例 1：**

<pre><strong>输入：</strong>intervals = [[1,3],[2,6],[8,10],[15,18]]
<strong>输出：</strong>[[1,6],[8,10],[15,18]]
<strong>解释：</strong>区间 [1,3] 和 [2,6] 重叠, 将它们合并为 [1,6].
</pre>

---

## 题解：排序预处理与动态边界合并策略

### 1. 核心思路：将二维问题转化为一维延伸

区间合并的核心难点在于区间出现的顺序是随机的。你的代码抓住了问题的关键：**先排序，后合并**。

- **排序消除维度约束**：通过 `intervals.sort(key=lambda x : x[0])`，我们保证了所有区间是按照“左端点”从小到大排列的。这样，我们只需要考虑当前区间与上一个合并区间的“右端点”关系，而不需要担心前方会出现起始位置更早的区间。

- **贪心合并**：维护一个当前正在合并的范围 `[l, r]`。遍历后续区间时，根据是否有重叠来决定是“合为一体”还是“另起炉灶”。

---

### 2. 执行逻辑的详细拆解

#### A. 排序基准

`intervals.sort(key=lambda x : x[0])`：排序后，对于任何相邻区间 $i$ 和 $i+1$，必然满足 $start_{i} \le start_{i+1}$。这意味着如果这两个区间不重叠，唯一的可能就是 $start_{i+1} > end_i$。

#### B. 遍历与状态转移

1. **初始化**：取第一个区间作为当前的合并基准 `l, r`。

2. **遇到不重叠区间（`new_l > r`）**：
   
   - 说明当前的 `[l, r]` 已经到头了，无法再向右延伸。
   
   - 将 `[l, r]` 存入结果 `ans`。
   
   - 更新 `l = new_l`，开启一个新的待合并区间。

3. **遇到重叠或包含区间**：
   
   - 只要 `new_l <= r`，就说明两个区间有交集。
   
   - **更新右边界**：`r = max(r, new_r)`。这里使用 `max` 非常关键，因为可能存在大区间完全包裹小区间的情况（例如 `[1, 10]` 和 `[2, 5]`）。

#### C. 收尾处理

循环结束后，最后一组 `[l, r]` 还在内存中尚未存入列表，因此需要手动执行 `ans.append([l, r])`。

---

### 3. 逻辑亮点：简洁的边界更新

代码中 `r = max(r, new_r)` 放在了 `if` 判断之外（或者说在逻辑流的下游），这非常巧妙。

- 如果发生了 `new_l > r`，`l` 更新为 `new_l` 后，下一行 `r = max(r, new_r)` 会直接把 `r` 也重置为 `new_r`。

- 如果没有发生 `new_l > r`，则执行正常的右边界扩张。
  
  这种写法避免了冗余的 `else` 分支，使逻辑更加线性。

---

### 4. 复杂度分析

- **时间复杂度**：$O(N \log N)$。主要的开销在于排序，随后的线性扫描只需 $O(N)$。

- **空间复杂度**：$O(\log N)$ 或 $O(N)$。这取决于排序算法的实现（Python 的 Timsort 通常需要一定的额外空间），结果数组 `ans` 不计入算法空间复杂度。

---

### 代码回顾

Python

```
class Solution:
    def merge(self, intervals: List[List[int]]) -> List[List[int]]:
        # 1. 按左端点排序
        intervals.sort(key=lambda x : x[0])
        n = len(intervals)
        ans = []

        # 2. 初始化第一个区间
        l, r = intervals[0][0], intervals[0][1]

        for i in range(1, n):
            new_l, new_r = intervals[i][0], intervals[i][1]

            # 3. 判定是否产生了“断层”
            if new_l > r:
                ans.append([l, r]) # 存入上一个已完成合并的区间
                l = new_l          # 重置左起点

            # 4. 无论是否断层，都尝试更新最右边界
            r = max(r, new_r)

        # 5. 处理最后一个留在内存中的区间
        ans.append([l, r])
        return ans
```
