## [71. 简化路径](https://leetcode.cn/problems/simplify-path/)

### 中等

给你一个字符串 `path` ，表示指向某一文件或目录的 Unix 风格 **绝对路径** （以 `'/'` 开头），请你将其转化为 **更加简洁的规范路径**。

在 Unix 风格的文件系统中规则如下：

- 一个点 `'.'` 表示当前目录本身。
- 此外，两个点 `'..'` 表示将目录切换到上一级（指向父目录）。
- 任意多个连续的斜杠（即，`'//'` 或 `'///'`）都被视为单个斜杠 `'/'`。
- 任何其他格式的点（例如，`'...'` 或 `'....'`）均被视为有效的文件/目录名称。

返回的 **简化路径** 必须遵循下述格式：

- 始终以斜杠 `'/'` 开头。
- 两个目录名之间必须只有一个斜杠 `'/'` 。
- 最后一个目录名（如果存在）**不能** 以 `'/'` 结尾。
- 此外，路径仅包含从根目录到目标文件或目录的路径上的目录（即，不含 `'.'` 或 `'..'`）。

返回简化后得到的 **规范路径** 。

**示例 1：**

<p><span class="example-io"><b>输出：</b>"/home"</span></p>

<p><strong>解释：</strong></p>

## 题解：栈（Stack）在文件系统导航中的模拟应用

### 1. 核心思路：路径的“进”与“退”

简化 Unix 路径的核心在于处理目录切换逻辑。我们可以将路径看作一系列的操作指令，而栈（Stack）是模拟这种“进入目录”和“返回上级”逻辑的最自然选择。

你的代码采用了**分治处理**的策略：

- **预处理**：利用 `split('/')` 将复杂的字符串切分为单纯的“指令”列表。

- **进栈（进入目录）**：遇到有效的目录名，将其压入栈中。

- **出栈（返回上级）**：遇到 `..`，将栈顶元素弹出。

- **忽略（原地不动）**：遇到 `.` 或由多余斜杠生成的空字符串，不做任何操作。

---

### 2. 执行逻辑的详细拆解

#### A. 字符串拆解

`directory_list = path.split('/')`

这一步非常巧妙地解决了“连续斜杠”的问题。例如，路径 `"/home//foo/"` 会被切分为 `['', 'home', '', 'foo', '']`。在后续处理中，我们只需要过滤掉空字符串即可。

#### B. 核心状态机逻辑

遍历拆分后的 `directory_list`，对于每个 `directory`：

1. **处理 `..`（向上级跳跃）**：
   
   - 如果栈不为空，执行 `stack.pop()`。这代表回退到父目录。
   
   - 如果栈为空（已经在根目录），则忽略它，因为根目录的父目录还是根目录。

2. **处理 `.` 和空字符串**：
   
   - `elif directory != '.' and directory != ''`：
   
   - 这两类指令代表“留在原地”，逻辑上不需要对栈做任何改变，直接跳过。

3. **处理有效目录名**：
   
   - 剩下的任何字符串（包括题目提到的 `...` 等）都被视为文件夹名称，执行 `stack.append(directory)`。

#### C. 路径规范化重建

`'/' + '/'.join(stack)`

最后，利用斜杠将栈内的目录名重新连接起来，并在开头补上绝对路径的标志 `/`。这种方式自动处理了“结尾不能有斜杠”的约束（除非结果就是根目录本身）。

---

### 3. 示例模拟

以路径 `path = "/a/./b/../../c/"` 为例：

1. `split` 得到：`['', 'a', '.', 'b', '..', '..', 'c', '']`

2. 遇到 `a`：栈 `['a']`

3. 遇到 `.`：忽略

4. 遇到 `b`：栈 `['a', 'b']`

5. 遇到第一个 `..`：弹出 `b`，栈 `['a']`

6. 遇到第二个 `..`：弹出 `a`，栈 `[]`

7. 遇到 `c`：栈 `['c']`

8. 结果：`'/' + 'c' = "/c"`

---

### 4. 复杂度分析

- **时间复杂度**：$O(N)$。其中 $N$ 是路径字符串的长度。拆分字符串需要 $O(N)$，遍历处理列表同样需要 $O(N)$。

- **空间复杂度**：$O(N)$。在最坏情况下（没有任何 `..` 或 `.`），栈和拆分后的列表都需要存储所有的目录名。

---

### 代码回顾

```python
class Solution:
    def simplifyPath(self, path: str) -> str:
        stack = []
        # 1. 拆分路径，过滤掉多余的斜杠
        directory_list = path.split('/')

        for directory in directory_list:
            # 2. 处理返回上级目录
            if directory == '..':
                if stack:
                    stack.pop()
            # 3. 排除当前目录标识 '.' 和空字符串
            elif directory != '.' and directory != '':
                # 4. 有效目录名，入栈
                stack.append(directory)

        # 5. 组装规范路径
        return '/' + '/'.join(stack)
```
