## [73. 矩阵置零](https://leetcode.cn/problems/set-matrix-zeroes/)

### 中等

给定一个 *m* x *n*的矩阵，如果一个元素为 **0** ，则将其所在行和列的所有元素都设为 **0** 。请使用 **[原地](http://baike.baidu.com/item/%E5%8E%9F%E5%9C%B0%E7%AE%97%E6%B3%95)** 算法。

## 题解：首列标志位与倒序覆盖的最优原地算法

### 1. 核心思路：极简标记策略

在 $O(1)$ 额外空间的限制下，最优雅的解法是直接利用矩阵的**第一行**和**第一列**作为“存根”，记录对应行或列是否需要清零。你的代码在此基础上更进一步，通过引入一个变量 `flag_col0`，将空间优化到了极致：

- **第一行（`matrix[0][...]`）**：记录第 $j$ 列是否包含 $0$。

- **第一列（`matrix[...][0]`）**：记录第 $i$ 行是否包含 $0$。

- **关键冲突处理**：由于 `matrix[0][0]` 既属于第一行也属于第一列，会产生逻辑覆盖。你的代码巧妙地用 `matrix[0][0]` 仅代表**第一行**的状态，而额外使用 `flag_col0` 独立记录**第一列**的状态。

---

### 2. 执行逻辑的详细拆解

#### A. 扫描与标记（前序遍历）

代码首先遍历整个矩阵：

1. **首列特殊处理**：如果第一列的某个元素为 $0$，记录 `flag_col0 = True`。

2. **内部标记**：从第二列（`j=1`）开始扫描。只要发现 `matrix[i][j] == 0`，就将该行行首 `matrix[i][0]` 和该列列首 `matrix[0][j]` 均设为 $0$。

#### B. 倒序覆盖（关键逻辑点）

在根据标记位修改原数据时，代码采用了从后往前（倒序）的更新方式：

- **为什么要倒序？** 因为第一行和第一列包含了所有的“账本信息”。如果我们先修改第一行，那么第一行原本记录的关于“哪些列该清零”的信息就会丢失。

- **逻辑分步**：在每一行的内部，先处理 `j=1` 到 `n-1` 的列元素。由于此时 `matrix[i][0]` 和 `matrix[0][j]` 还没被修改，可以安全使用。

#### C. 首列终结处理

在处理完每一行的内部元素后，立即根据 `flag_col0` 决定是否将该行的行首 `matrix[i][0]` 置零。这种**随行处理**的方式非常高效。

---

### 3. 逻辑亮点：单变量解决重叠问题

普通的解法通常需要两个布尔变量分别记录首行和首列。你的代码通过将“内部列”的循环起点设为 `1`，并将首行状态寄存在 `matrix[0][0]` 中，从而**仅用一个额外变量 `flag_col0`** 就理清了复杂的覆盖关系。

---

### 4. 复杂度分析

- **时间复杂度**：$O(M \times N)$。矩阵的所有元素被遍历了两轮。

- **空间复杂度**：$O(1)$。除了一个布尔标志位，完全利用了原矩阵的存储空间。
