## [125. 验证回文串](https://leetcode.cn/problems/valid-palindrome/)

### 简单

<p>如果在将所有大写字符转换为小写字符、并移除所有非字母数字字符之后，短语正着读和反着读都一样。则可以认为该短语是一个 <strong>回文串</strong> 。</p>
<p>字母和数字都属于字母数字字符。</p>
<p>给你一个字符串 <code>s</code>，如果它是 <strong>回文串</strong> ，返回 <code>true</code><em> </em>；否则，返回<em> </em><code>false</code><em> </em>。</p>
<p> </p>
<p><strong>示例 1：</strong></p>
<pre><strong>输入:</strong> s = "A man, a plan, a canal: Panama"
<strong>输出：</strong>true
<strong>解释：</strong>"amanaplanacanalpanama" 是回文串。
</pre>

---

## 题解：过滤预处理 + 双指针法

### 1. 核心思路：标准化与首尾对撞

验证回文串的关键在于**排除干扰项**（忽略大小写、非字母数字字符）并**对称比较**。

你的代码采用了先“瘦身”再“比对”的策略：

- **标准化（清洗数据）**：先将原始字符串处理成一个纯粹的、仅包含小写字母和数字的新字符串。

- **双指针（对撞检查）**：在新字符串的两头设置指针，向中间靠拢，逐一验证字符是否对称。

### 2. 代码逻辑拆解

- **字符过滤与转换**：
  
  利用 Python 的 `isalnum()` 方法识别字母和数字，结合 `lower()` 将所有字符统一为小写。通过列表推导式和 `"".join()` 快速构建出一个剔除了所有空格、标点符号的纯净字符串 `new_s`。

- **初始化双指针**：
  
  设置左指针 `l = 0` 和右指针 `r = n - 1`，分别指向清洗后字符串的最前端和最后端。

- **对撞循环比对**：
  
  - 使用 `while l < r` 作为循环条件，保证指针在相遇前停止。
  
  - **失配检查**：如果在任何时刻 `new_s[l]` 与 `new_s[r]` 不相等，说明字符串不对称，立即判定为 `False`。
  
  - **步进**：如果匹配成功，则左指针右移、右指针左移，继续检查下一对字符。

- **最终判定**：
  
  如果循环顺利结束没有触发返回，说明所有对称位置的字符都一致，返回 `True`。

### 3. 复杂度分析

- **时间复杂度：$O(n)$**
  
  - 过滤字符串需要遍历一遍原字符串 $s$，耗时 $O(n)$。
  
  - 双指针比对最多遍历一遍处理后的字符串，耗时 $O(n)$。
  
  - 整体复杂度与字符串长度成线性关系。

- **空间复杂度：$O(n)$**
  
  - 代码创建了一个新的字符串 `new_s` 来存储过滤后的结果，在最坏情况下（原字符串全是字母数字）需要 $O(n)$ 的空间。
