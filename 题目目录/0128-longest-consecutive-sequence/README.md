## [128. 最长连续序列](https://leetcode.cn/problems/longest-consecutive-sequence/)

### 中等

给定一个未排序的整数数组 `nums` ，找出数字连续的最长序列（不要求序列元素在原数组中连续）的长度。

请你设计并实现时间复杂度为 `O(n)` 的算法解决此问题。

**示例 1：**

<pre><strong>输入：</strong>nums = [100,4,200,1,3,2]
<strong>输出：</strong>4
<strong>解释：</strong>最长数字连续序列是 [1, 2, 3, 4]。它的长度为 4。</pre>

---

## 题解：哈希集合与“逻辑起点”检测算法

### 1. 核心思路：从序列的“头”开始探索

本题要求在 $O(n)$ 时间内解决，这意味着我们不能使用排序（$O(n \log n)$）。你采用的 **哈希集合（Set）** 方案是该问题的经典最优解。

其核心智慧在于**去重**与**起点识别**：

- **去重**：首先将数组转化为集合 `num_set`，这样不仅消除了重复数字的干扰，还能实现 $O(1)$ 级别的快速查找。

- **寻找逻辑起点**：对于集合中的每一个数 `num`，我们只在它是**序列起点**时才开始计数。如何判断它是起点？只要 `num - 1` 不在集合中，它就是这个连续序列的“头”。

---

### 2. 执行逻辑的详细拆解

#### A. 过滤无效尝试

在循环 `for num in num_set` 中，`if num - 1 not in num_set` 是整段代码的“灵魂”：

- 如果 `num - 1` 存在，说明当前 `num` 只是某个长序列中间的一个点。我们跳过它，因为那个长序列会被它的起点处理到。

- 这样确保了每个连续序列只会被**完整地遍历一次**。

#### B. 序列扩张（内层循环）

一旦确认了起点，代码进入 `while` 循环：

1. 不断探测 `current_num + 1` 是否在集合中。

2. 通过 `current_len` 记录当前序列的长度。

3. 一旦序列断裂，停止探测。

#### C. 更新全局最值

利用 `max_len = max(max_len, current_len)` 记录在所有序列中找到的最长长度。

---

### 3. 逻辑亮点：如何达成 $O(n)$？

初看代码，`for` 循环嵌套 `while` 循环似乎是 $O(n^2)$，但实际上：

- 每个数字最多只会被“访问”两次：一次是在 `for` 循环中作为普通元素被检查，另一次是在 `while` 循环中作为某个序列的一员被累加。

- 对于不属于任何序列起点的数字，`while` 循环永远不会执行。

- 因此，总的操作次数与 $n$ 成线性关系。

---

### 4. 复杂度分析

- **时间复杂度**：$O(n)$。虽然有嵌套循环，但每个元素在 `while` 内部只会被遍历一次，摊还复杂度为线性。

- **空间复杂度**：$O(n)$。需要一个哈希集合来存储所有不重复的数字。

---

### 代码回顾

```python
class Solution:
    def longestConsecutive(self, nums: List[int]) -> int:
        # 1. 转化为集合，实现 O(1) 查找并去重
        num_set = set(nums)
        max_len = 0

        for num in num_set:
            # 2. 关键：只有当 num 是序列的起点时才开始计算
            if num - 1 not in num_set:
                current_num = num
                current_len = 1

                # 3. 顺着起点往后找
                while current_num + 1 in num_set:
                    current_num += 1
                    current_len += 1

                # 4. 更新最大长度
                max_len = max(max_len, current_len)

        return max_len
```
