## [141. 环形链表](https://leetcode.cn/problems/linked-list-cycle/)

### 中等

给你一个链表的头节点 `head` ，判断链表中是否有环。

如果链表中有某个节点，可以通过连续跟踪 `next` 指针再次到达，则链表中存在环。 为了表示给定链表中的环，评测系统内部使用整数 `pos` 来表示链表尾连接到链表中的位置（索引从 0 开始）。**注意：`pos` 不作为参数进行传递** 。仅仅是为了标识链表的实际情况。

*如果链表中存在环* ，则返回 `true` 。 否则，返回 `false` 。

## 题解：快慢指针（Floyd 判圈算法）

针对这段精简后的代码，其核心思想是利用**速度差**来检测链表中的环路。这种算法在空间复杂度上达到了极致的优化。

### 1. 算法核心：追及问题

在单向链表中，如果存在环，那么从某个节点开始，所有的 `next` 指针最终都会回到环内。

- **慢指针 (`slow`)**：相当于乌龟，每次移动 **1** 个节点。

- **快指针 (`fast`)**：相当于兔子，每次移动 **2** 个节点。

**逻辑推导**：

- **无环状态**：兔子速度快，会率先到达终点（指向 `None`），此时游戏结束，返回 `False`。

- **有环状态**：乌龟和兔子先后进入环中。由于兔子相对于乌龟的速度是 **1 节点/步**，它们之间的距离每移动一步就会缩短 1。因此，兔子一定会在环内某个位置从后方“套圈”乌龟。当 `slow == fast` 时，证明有环，返回 `True`。

---

### 2. 执行逻辑拆解

1. **初始化**：
   
   将 `slow` 和 `fast` 同时指向头节点 `head`。虽然也可以让 `fast` 先走一步，但初始化在同一点可以更整洁地处理空链表的情况。

2. **条件循环**：
   
   `while fast and fast.next:`
   
   - 这个条件保证了快指针及其下一步都是有效的。如果 `fast` 或 `fast.next` 为空，说明已经撞到了链表的尽头，链表无环。

3. **步进逻辑**：
   
   - `slow = slow.next`：慢指针平稳移动。
   
   - `fast = fast.next.next`：快指针跨步移动。

4. **碰撞检测**：
   
   `if slow == fast:`
   
   - 这是算法的终点。如果两者指向了同一个内存地址，说明快指针已经在环内追上了慢指针。

---

### 3. 为什么这是最优解？

- **空间复杂度：$O(1)$**
  
  我们只额外创建了两个指针变量。相比于使用哈希表（Set）存储已访问节点需要 $O(N)$ 的空间，快慢指针完全不需要额外内存。

- **时间复杂度：$O(N)$**
  
  - **情况 A（无环）**：快指针走完链表全长所需时间与 $N$ 成正比。
  
  - **情况 B（有环）**：慢指针进入环后，快指针在接下来的一圈内必然会追上慢指针。总步数不会超过链表节点总数。

---

### 4. 代码回顾

```python
class Solution:
    def hasCycle(self, head: Optional[ListNode]) -> bool:
        # 初始化双指针
        slow = fast = head

        # 快指针能走两步的前提是当前和下一步都不为空
        while fast and fast.next:
            slow = slow.next          # 走 1 步
            fast = fast.next.next     # 走 2 步

            # 如果相遇，则一定有环
            if slow == fast:
                return True

        # 正常走完循环，说明快指针遇到了终点，无环
        return False
```
