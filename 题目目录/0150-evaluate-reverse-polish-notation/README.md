## [150. 逆波兰表达式求值](https://leetcode.cn/problems/evaluate-reverse-polish-notation/)

### 中等

给你一个字符串数组 `tokens` ，表示一个根据 [逆波兰表示法](https://baike.baidu.com/item/%E9%80%86%E6%B3%A2%E5%85%B0%E5%BC%8F/128437) 表示的算术表达式。

请你计算该表达式。返回一个表示表达式值的整数。

**注意：**

- 有效的算符为 `'+'`、`'-'`、`'*'` 和 `'/'` 。
- 每个操作数（运算对象）都可以是一个整数或者另一个表达式。
- 两个整数之间的除法总是 **向零截断** 。
- 表达式中不含除零运算。
- 输入是一个根据逆波兰表示法表示的算术表达式。
- 答案及所有中间计算结果可以用 **32 位** 整数表示。

**示例 1：**

<pre><strong>输入：</strong>tokens = ["2","1","+","3","*"]
<strong>输出：</strong>9
<strong>解释：</strong>该算式转化为常见的中缀算术表达式为：((2 + 1) * 3) = 9
</pre>

## 题解：栈与函数映射的逆波兰计算策略

### 1. 核心思路：后缀表达式的线性求值

逆波兰表示法（RPN）是一种不需要括号来标识运算优先级的数学表达式。它的求值规则非常适合使用 **栈（Stack）** 来处理：

- **遇到数字**：说明它是操作数，将其入栈。

- **遇到算符**：说明需要执行运算。从栈中弹出两个数字，进行运算后将结果重新压入栈中。

- **结果**：遍历结束后，栈顶剩下的唯一元素就是最终答案。

你的代码巧妙地结合了 **Python 函数映射** 和 **异常处理机制**，使代码逻辑非常紧凑且具有扩展性。

---

### 2. 执行逻辑的详细拆解

#### A. 算符函数映射

`op_to_binary_fn` 字典将字符算符映射为具体的运算函数。

- **向零截断的除法**：这是本题的一个难点。Python 的 `//` 是向下取整（例如 `-1 // 10 = -1`），而题目要求向零截断（即 `-1 / 10 = -0.1` 取整为 `0`）。你的 `lambda x, y: int(x / y)` 完美解决了这个问题，因为 `int()` 函数在转换浮点数时会自动向零取整。

#### B. 异常驱动的类型识别

代码使用了 `try-except` 结构来区分“数字”和“算符”：

1. **数字入栈**：尝试 `int(token)`。如果转换成功，说明是数字，存入 `num`。

2. **算符运算**：如果抛出 `ValueError`，说明当前 token 是算符：
   
   - **弹出顺序**：先弹出的作为右操作数 `num2`，后弹出的作为左操作数 `num1`（减法和除法对顺序敏感）。
   
   - **计算结果**：从字典调用对应函数计算出 `num`。

#### C. 统一入栈逻辑

`finally: stack.append(num)` 保证了无论是直接读取的数字，还是计算出的结果，最终都会被压入栈中供下一步使用。

---

### 3. 复杂度分析

- **时间复杂度**：$O(N)$。只需遍历一次字符串数组，每个元素入栈出栈各一次，哈希表查询和算术运算均为 $O(1)$。

- **空间复杂度**：$O(N)$。在最坏情况下（例如数组全是数字），栈需要存储所有的操作数。

---

### 4. 代码回顾

```python
class Solution:
    def evalRPN(self, tokens: List[str]) -> int:
        op_to_binary_fn = {
            "+": add,
            "-": sub,
            "*": mul,
            "/": lambda x, y: int(x / y),
        }

        stack = []
        for token in tokens:
            try:
                num = int(token)
            except ValueError:
                num2 = stack.pop()
                num1 = stack.pop()
                num = op_to_binary_fn[token](num1, num2)
            finally:
                stack.append(num)

        return stack[0]
        
```
