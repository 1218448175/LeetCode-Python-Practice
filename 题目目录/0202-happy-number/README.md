## [202. 快乐数](https://leetcode.cn/problems/happy-number/)

### 简单

编写一个算法来判断一个数 `n` 是不是快乐数。

**「快乐数」** 定义为：

- 对于一个正整数，每一次将该数替换为它每个位置上的数字的平方和。
- 然后重复这个过程直到这个数变为 1，也可能是 **无限循环** 但始终变不到 1。
- 如果这个过程 **结果为** 1，那么这个数就是快乐数。

如果 `n` 是 *快乐数* 就返回 `true` ；不是，则返回 `false` 。

**示例 1：**

<pre><strong>输入：</strong>n = 19
<strong>输出：</strong>true
<strong>解释：
</strong>1<sup>2</sup> + 9<sup>2</sup> = 82
8<sup>2</sup> + 2<sup>2</sup> = 68
6<sup>2</sup> + 8<sup>2</sup> = 100
1<sup>2</sup> + 0<sup>2</sup> + 0<sup>2</sup> = 1
</pre>

## 题解：哈希集合寻路与循环检测策略

### 1. 核心思路：捕捉“无限循环”

判断一个数是否为快乐数，本质上是在追踪一个数值序列的演化路径。根据题目定义，演化结果只有两种可能：

1. **收敛**：数值最终变为 $1$。

2. **发散/循环**：数值进入一个**不包含 1 的死循环**。

由于 $2^{31}-1$ 范围内的数经过平方和计算后会迅速缩小并稳定在一个较小的范围内，我们不需要担心数值会无限增大。因此，问题的关键在于：**如何判断我们是否回到了曾经走过的路？**

你的代码采用了 **哈希集合（`set`）** 来记录路径上出现过的所有数字。一旦新生成的数字在集合中已存在，说明我们进入了循环，可以断定它永远不会到达 1。

---

### 2. 执行逻辑的详细拆解

#### A. 位平方和函数 (`check_sum`)

这是逻辑的基石。通过取模 (`% 10`) 获取个位数，通过整除 (`// 10`) 剥离高位数，循环累加每个位上的平方值。

#### B. 路径追踪与碰撞检测

在 `while True` 循环中：

1. **计算演化**：调用 `check_sum` 获取下一个数 `cnt`。

2. **成功判定**：如果 `cnt == 1`，说明找到了通往快乐的钥匙，直接返回 `True`。

3. **循环判定（核心）**：
   
   - 如果当前的 `num` 还没有出现在 `num_set` 中，将其存入集合，继续下一次演化。
   
   - 如果 `num` 已经存在于集合中，说明此时**已经撞上了之前的路径**，闭环已成，立即返回 `False`。

#### C. 状态更新

将 `num` 更新为 `cnt`，开启下一轮迭代。

---

### 3. 逻辑亮点：数学上的必然性

这种方法非常稳健，因为：

- **空间换时间**：利用集合 $O(1)$ 的查找效率，能够极快地识别出循环。

- **终止保证**：数学证明显示，不快乐数最终都会进入一个固定的循环（如包含 4, 16, 37, 58, 89, 145, 42, 20 的循环），你的代码逻辑完美捕捉了这一特征。

---

### 4. 复杂度分析

- **时间复杂度**：$O(\log N)$。对于数字 $N$，其位平方和的下降速度极快，随后进入循环或收敛的步骤相对固定，整体复杂度与 $N$ 的位数相关。

- **空间复杂度**：$O(\log N)$。哈希集合中存储的中间结果数量与数值演化的跨度有关，对于给定的整型范围，这是一个非常小的额外空间开销。

---

### 代码回顾

```python
def check_sum(n):
    cnt = 0
    while n > 0:
        cnt += (n % 10) ** 2
        n = n // 10
    return cnt

class Solution:
    def isHappy(self, n: int) -> bool:
        num_set = set()
        num = n
        while True:
            cnt = check_sum(num)
            if cnt == 1:
                return True
            # 关键判定：如果当前数字再次出现，说明进入死循环
            if num in num_set:
                return False

            num_set.add(num)
            num = cnt
```

> **小技巧**：其实这种“检测循环”的场景，除了使用哈希集合，还可以使用**快慢指针（弗洛伊德循环查找算法）**，将空间复杂度进一步优化到 $O(1)$ 哦！进阶代码如下：

```python
def check_sum(n):
    cnt = 0
    while n > 0:
        cnt += (n % 10) ** 2
        n = n // 10
    return cnt

class Solution:
    def isHappy(self, n: int) -> bool:
        slow = fast = n
        while True:
            slow = check_sum(slow)
            fast = check_sum(check_sum(fast)
            if slow == fast:
                break
        return True if slow == 1 else False
```
