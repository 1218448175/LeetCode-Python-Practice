## [289. 生命游戏](https://leetcode.cn/problems/game-of-life/)

### 中等

根据 [百度百科](https://baike.baidu.com/item/%E7%94%9F%E5%91%BD%E6%B8%B8%E6%88%8F/2926434?fr=aladdin) ， **生命游戏** ，简称为 **生命** ，是英国数学家约翰·何顿·康威在 1970 年发明的细胞自动机。

给定一个包含 `m × n` 个格子的面板，每一个格子都可以看成是一个细胞。每个细胞都具有一个初始状态： `1` 即为 **活细胞** （live），或 `0` 即为 **死细胞** （dead）。每个细胞与其八个相邻位置（水平，垂直，对角线）的细胞都遵循以下四条生存定律：

1. 如果活细胞周围八个位置的活细胞数少于两个，则该位置活细胞死亡；
2. 如果活细胞周围八个位置有两个或三个活细胞，则该位置活细胞仍然存活；
3. 如果活细胞周围八个位置有超过三个活细胞，则该位置活细胞死亡；
4. 如果死细胞周围正好有三个活细胞，则该位置死细胞复活；

下一个状态是通过将上述规则同时应用于当前状态下的每个细胞所形成的，其中细胞的出生和死亡是 **同时** 发生的。给你 `m x n` 网格面板 `board` 的当前状态，返回下一个状态。

给定当前 `board` 的状态，**更新** `board` 到下一个状态。

**注意** 你不需要返回任何东西。

## 题解：复合状态编码与原地演化策略

### 1. 核心思路：状态叠加与原地修改

生命游戏要求所有细胞的状态更新是“同时”发生的。直接原地修改会导致后续细胞在计数时读到的是“新状态”而非“初始状态”，从而产生错误。

为了实现 **$O(1)$ 额外空间** 的要求，我们利用细胞只有 `0` 和 `1` 两个原始状态的特性，通过**复合状态编码**来记录细胞的演化轨迹：

- **状态 0**：初始死，后续仍死。

- **状态 1**：初始活，后续仍活。

- **状态 2**：初始活，更新后死亡（1 $\to$ 0）。

- **状态 3**：初始死，更新后复活（0 $\to$ 1）。

通过这种编码，我们在计数时只要判断 `abs(board[i][j]) % 2 == 1`（或判断状态是否为 1 或 2），就能找回其“初始活细胞”的身份。

---

### 2. 执行逻辑的详细拆解

#### A. 八连通区域计数

对于每一个细胞 $(i, j)$，我们遍历其周围 8 个邻居。为了代码简洁，可以使用位移数组或双层循环偏移量 `[-1, 0, 1]`。在统计邻居活细胞数量时，关键在于识别那些**当前虽变为状态 2 但初始为活**的细胞。

#### B. 规则映射与编码

根据周围活细胞总数 `live_neighbors` 实施生存定律：

- **活细胞（状态 1）**：
  
  - 若数量 $< 2$ 或 $> 3$，标记为 `2`（即将死亡）。
  
  - 若数量为 $2$ 或 $3$，保持 `1`。

- **死细胞（状态 0）**：
  
  - 若数量 $== 3$，标记为 `3`（即将复活）。

#### C. 状态最终还原

当整个矩阵都被编码完成后，再次遍历矩阵，将临时中间态还原为最终的生命状态：

- `2` $\to$ `0`

- `3` $\to$ `1`

- `0` 和 `1` 保持不变。

---

### 3. 逻辑亮点：利用模运算/绝对值的自愈性

由于 `2` 是从 `1` 变来的，`3` 是从 `0` 变来的，我们可以发现一个数学规律：

- **原始状态为活**的编码（1 和 2）在 `mod 2` 之后均为 `1`（或者在你的代码逻辑中通过位运算处理）。这种设计让我们在一次遍历中既存储了“过去”，又预告了“未来”。

---

### 4. 复杂度分析

- **时间复杂度**：$O(M \times N)$。矩阵中的每个元素被遍历两次（一次编码，一次还原），邻居扫描是常数级（8 个方向）。

- **空间复杂度**：$O(1)$。除了几个循环变量外，没有使用任何额外空间，完全在原数组上操作。
