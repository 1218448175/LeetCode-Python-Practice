## [452. 用最少数量的箭引爆气球](https://leetcode.cn/problems/minimum-number-of-arrows-to-burst-balloons/)

### 中等

有一些球形气球贴在一堵用 XY 平面表示的墙面上。墙面上的气球记录在整数数组 `points` ，其中`points[i] = [xstart, xend]` 表示水平直径在 `xstart` 和 `xend`之间的气球。你不知道气球的确切 y 坐标。

一支弓箭可以沿着 x 轴从不同点 **完全垂直** 地射出。在坐标 `x` 处射出一支箭，若有一个气球的直径的开始和结束坐标为 `xstart`，`xend`， 且满足  `xstart ≤ x ≤ xend`，则该气球会被 **引爆** 。可以射出的弓箭的数量 **没有限制** 。 弓箭一旦被射出之后，可以无限地前进。

给你一个数组 `points` ，*返回引爆所有气球所必须射出的 **最小** 弓箭数* 。

 

**示例 1：**

<pre><strong>输入：</strong>points = [[10,16],[2,8],[1,6],[7,12]]
<strong>输出：</strong>2
<strong>解释：</strong>气球可以用2支箭来爆破:
-在x = 6处射出箭，击破气球[2,8]和[1,6]。
-在x = 11处发射箭，击破气球[10,16]和[7,12]。</pre>

---

## 题解：贪心算法与右端点排序策略

### 1. 核心思路：贪婪的射击点选择

这道题是经典的**区间覆盖问题**。为了用最少的箭引爆最多的气球，我们的核心策略是：**尽量让一支箭穿过重叠区域最多的地方。**

你的代码采用了一个非常巧妙的贪心策略：

- **以右端点排序**：将气球按照结束坐标 `xend` 从小到大排序。

- **射击点定在右端点**：对于当前的第一个气球，为了让这支箭尽可能地影响到后面的气球，我们将箭射在它的“最右侧边缘”（即 `pos = points[i][1]`）。

- **跳过已引爆气球**：后续只要气球的起始位置 `xstart` 小于或等于当前的 `pos`，这支箭就一定能顺带引爆它。

- **更新射击点**：一旦遇到一个气球的 `xstart > pos`，说明当前的箭够不到了，必须使用一支新箭，并更新射击点为这个新气球的右端点。

---

### 2. 执行逻辑的详细拆解

#### A. 排序的重要性

`points.sort(key=lambda x: x[1])`：

为什么按右端点排序？因为右端点决定了一个气球**必须被引爆的最后期限**。如果我们按照左端点排序，虽然也可以解题，但逻辑会复杂一些（需要动态缩小重叠区间的右边界）。按右端点排序能保证我们每次选取的射击点都是当前最迫切需要处理的边界。

#### B. 贪心迭代

1. **初始化**：第一支箭射在第一个气球的末尾 `pos = points[0][1]`，计数 `ans = 1`。

2. **遍历对比**：
   
   - `start = points[i][0]`。
   
   - **判定**：如果 `start > pos`，说明当前气球与上一支箭覆盖的范围完全断开了。
   
   - **动作**：增加箭的数量，并将射击点 `pos` 更新为当前气球的末尾，因为它成为了下一组重叠区间的“新限制”。

---

### 3. 逻辑亮点：为什么这样是最优的？

这是一种**局部最优导致全局最优**的典型案例。每一支箭都射在“当前能覆盖到最多后续气球”的位置（即当前已知重叠区间的共同右边界）。由于我们是按右端点排序的，这种做法保证了不会有任何一个气球被遗漏，同时也最大限度地利用了每支箭的潜力。

---

### 4. 复杂度分析

- **时间复杂度**：$O(N \log N)$。主要消耗在排序上，随后的遍历是 $O(N)$。

- **空间复杂度**：$O(\log N)$。排序所需的栈空间。

---

### 5. 极简代码回顾

```python
class Solution:
    def findMinArrowShots(self, points: List[List[int]]) -> int:
        points.sort(key=lambda x: x[1])
        pos = points[0][1]
        n = len(points)
        i = 0
        ans = 1
        while i < n:
            start = points[i][0]
            if start > pos:
                pos = points[i][1]
                ans += 1
            i += 1
        return ans
```
